# Panda Mobile 合规清单 (V0.6)

## 1. 当前版本范围

MVP 仅覆盖：
1. 手机号 + 验证码登录（mock）
2. 登录成功进入缘分墙活跃态页面
3. 缘分墙会员入口灰态展示（不可点击）
4. 缘分墙动作按钮：`再等等` / `感兴趣`（mock 决策）
5. mock 匹配成功后跳转消息栏占位页

当前不覆盖：
1. 真实聊天能力
2. 举报/拉黑
3. 账号删除
4. 非 MVP 聊天组件接入（允许保留文件，但主流程禁止 import/路由跳转引用）

## 2. 登录与数据安全

1. 验证码错误提示文案固定为 `验证码错误`。
2. 日志不得记录完整手机号与验证码。
3. MVP 不使用登录鉴权 token，不做登录态持久化（UI 仍需使用 design tokens）。
4. 登录成功用户信息仅保存在内存，不落盘；仅在 App 进程被杀后清空。
5. 错误弹窗 3 秒内全屏完全禁交互。
6. 同一自然日内仅展示 1 个推荐对象，点击动作后不刷新下一张。
7. 同一自然日内重新登录，仍展示同一推荐对象。
8. 当天动作点击即锁定，不允许改选（`再等等` 与 `感兴趣` 互斥）。
9. 当天动作锁定后再次点击为静默忽略（不弹 toast）。
10. App 进程被杀后动作锁定不保留。
11. 日期边界按设备本地自然日（00:00）计算，并在 00:00 自动重置当日状态。
12. 页面停留跨越 00:00 时立即重置并换到历史未推荐卡片，不等待下次交互/onShow。
13. 推荐对象按用户循环历史去重：当前循环内已推荐对象不允许再次推荐给该用户。
14. 未推荐池耗尽时当日保持当前卡片，下一天 00:00 重置循环历史并重新循环。
15. 次日 00:00 进入新循环后，首个推荐对象优先不等于上一自然日最后展示对象；无可选候选可降级重复。
16. “上一自然日最后展示对象”按系统最后一次成功分配推荐对象定义。
17. 长期历史仅在卸载/清除应用数据时清空。

## 3. 协议与隐私入口（MVP）

1. 登录页需展示用户协议/隐私政策入口。
2. 当前点击行为使用 toast 占位，不做真实跳转：
   - 用户协议：`用户协议建设中`
   - 隐私政策：`隐私政策建设中`
3. 协议/隐私 toast 1500ms 内不重复弹出，且两者各自独立节流。

## 4. 审核准备（MVP）

1. 提供 mock 测试码说明：`123456`。
2. 提供错误场景说明：输入非 `123456` 展示验证码错误弹窗并 3 秒关闭。
3. 从启动到进入缘分墙活跃态路径可重复验证。
4. mock 池文件 `panda-mobile/mock/affinity-profiles.mock.json` 至少含 3 条资料。
5. 登录与缘分墙 `loading` 当前默认最小时长为 600ms（可按体验调整）。
6. `再等等` / `感兴趣` 为 mock 行为，需提供“非真实匹配/非真实聊天”测试说明。
7. `再等等` 提示文案固定：`已为你安排明日推荐`。
8. `感兴趣` 未匹配时提示文案固定：`今日匹配已用完，请明日再来`。
9. 消息占位页返回应按当前 `dateKey` 回到缘分墙应展示卡片（立即重算）。
10. `matched=true` 时 `conversationId` 生成规则固定为 `profileId + "_" + dateKey`。
11. 消息占位页在 MVP 阶段不处理跨 00:00 特殊逻辑。
12. `login_submit_blocked` 在首次进入登录页初始非法态不触发。

## 5. 发布前检查

1. `npm run ui:check` 通过。
2. 登录成功与失败路径真机验证通过。
3. 登录成功后跳转缘分墙活跃态成功率满足目标。
