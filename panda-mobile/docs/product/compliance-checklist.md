# Panda Mobile 合规清单 (V0.2)

## 1. 当前版本范围

MVP 仅覆盖：
1. 手机号 + 验证码登录（mock）
2. 登录成功进入缘分墙活跃态页面

当前不覆盖：
1. 聊天
2. 举报/拉黑
3. 账号删除

## 2. 登录与数据安全

1. 验证码错误提示文案固定为 `验证码错误`。
2. 日志不得记录完整手机号与验证码。
3. MVP 不使用 token，不做登录态持久化。

## 3. 协议与隐私入口（MVP）

1. 登录页需展示用户协议/隐私政策入口。
2. 当前可使用 mock 协议数据与占位链接：
   - `https://mock.local/terms`
   - `https://mock.local/privacy`

## 4. 审核准备（MVP）

1. 提供 mock 测试码说明：`123456`。
2. 提供错误场景说明：输入非 `123456` 展示验证码错误弹窗并 3 秒关闭。
3. 从启动到进入缘分墙活跃态路径可重复验证。

## 5. 发布前检查

1. `npm run ui:check` 通过。
2. 登录成功与失败路径真机验证通过。
3. 登录成功后跳转缘分墙活跃态成功率满足目标。
