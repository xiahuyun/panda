# Panda Mobile 合规清单（iOS 审核导向）

## 1. UGC / 社交类必备能力

1. 举报入口：聊天页可达
2. 拉黑入口：聊天页可达
3. 可联系信息：支持审核团队联系处理
4. 内容治理：基础审核策略与处置机制

## 2. 账号与隐私

1. App 内可删除账号（非仅网页）
2. 隐私政策 URL 可访问且内容完整
3. 数据最小化收集，敏感信息说明用途
4. 用户可请求删除个人数据（流程可执行）

## 3. 登录与第三方账号

1. 若接入第三方登录，需提供等价登录方案
2. 登录失败提示不可泄露敏感系统信息

## 4. 支付与商业化（若启用）

1. 数字内容/会员需走 IAP（StoreKit）
2. 不得绕过平台支付政策

## 5. 审核准备

1. 提供可用审核账号（含测试手机号/验证码方案）
2. 审核时后端稳定可访问
3. 提供测试路径说明文档（从启动到核心功能）

## 6. 发布前检查

1. `npm run ui:check` 通过
2. 主流程真机回归通过
3. 崩溃与网络异常处理可用
4. 举报、拉黑、账号删除入口均可点击到达
