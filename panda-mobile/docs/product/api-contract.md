# Panda Mobile Mock Contract (V0.5)

## 1. 当前约束

1. MVP **不接真实后端**，仅使用本地 mock service。
2. 登录不发放真实鉴权 token，不持久化登录状态。
3. 登录成功后仅在内存保存用户基础信息（当前进程有效）。
4. App 前后台切换不清空内存会话；仅在 App 进程被杀后清空。
5. 本文档是前端 mock 契约，后续联调前再产出正式 API 合同。

## 2. Mock 登录规则

前端预校验（调用 mock 前）：
1. `phone` 必须为 11 位数字；
2. `phone` 不限制号段，仅校验长度与数字格式；
3. `code` 必须为 6 位数字；
4. 不满足时，登录/注册按钮禁用，不发起 mock 校验。

输入：

```json
{
  "phone": "13800138000",
  "code": "123456"
}
```

校验规则：
1. `phone` 必须为 11 位数字；
2. `phone` 不限制号段，仅校验长度与数字格式；
3. `code` 必须为 6 位数字；
4. 测试通过码固定为 `123456`；
5. 其余情况统一视为失败：`验证码错误`。

成功返回（示例）：

```json
{
  "success": true,
  "data": {
    "userId": "mock_user_001",
    "nickname": "熊猫同学"
  }
}
```

内存会话写入（示例）：

```json
{
  "sessionUser": {
    "userId": "mock_user_001",
    "nickname": "熊猫同学"
  }
}
```

失败返回（示例）：

```json
{
  "success": false,
  "error": "验证码错误"
}
```

## 3. 登录失败弹窗规则

1. 弹窗文案：`验证码错误`。
2. 自动关闭时长：3 秒。
3. 弹窗期间锁定交互：全屏完全禁交互（不允许手动关闭、不允许重复提交、不可点击页面任意控件）。
4. 弹窗期间禁止输入框编辑，禁止系统返回手势/返回键关闭页面。
5. 关闭后停留登录页，允许再次输入并提交。

## 4. 缘分墙活跃态 Mock 数据

登录成功后进入「缘分墙 - 活跃状态 (普通用户版)」，使用以下字段：

```json
{
  "state": "active",
  "profile": {
    "id": "p_001",
    "name": "林静宜",
    "avatarUrl": "/static/logo.png",
    "meta": "26岁 · 165cm · 上海",
    "bio": "热爱生活中的小确幸...",
    "reason": "你们在兴趣和交流节奏上较匹配",
    "isOnline": true
  }
}
```

数据来源规则：
1. 本地维护 `profile` mock 池（最小规模 3 条）；
2. mock 池文件路径：`panda-mobile/mock/affinity-profiles.mock.json`；
3. 每次登录成功后从 mock 池随机选择 1 条展示；
4. 单次会话内不要求去重。
5. MVP 头像统一占位，使用 `/static/logo.png`。

## 5. 默认时长建议（MVP）

1. 登录校验 `loading` 当前默认时长：`600ms`（可按体验调整，建议避免闪烁）。
2. 缘分墙数据加载 `loading` 当前默认时长：`600ms`（可按体验调整，建议避免闪烁）。
3. 协议/隐私 toast 触发节流：`1500ms` 内不重复弹出，且用户协议/隐私政策各自独立计时。

## 6. 后续切换真实后端

1. 保持 service interface 不变，替换实现层。
2. 联调前补充正式 API Contract（错误码、鉴权、限流、幂等）。

## 7. MVP 辅助说明

1. 协议入口点击 toast：`用户协议建设中`。
2. 隐私入口点击 toast：`隐私政策建设中`。
3. 缘分墙活跃态本轮只实现成功数据路径，不实现错误态注入开关。
