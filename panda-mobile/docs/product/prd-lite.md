# Panda Mobile PRD Lite (V0.7)

## 1. 目标

本版本验证最小可用主流程（D5 P0）：
1. 用户进入「小熊交友 - 吃苹果熊猫 Logo 登录页」
2. 用户输入手机号与验证码并点击「登录/注册」
3. 前端先做输入合法性拦截（手机号 11 位数字，不限号段；验证码 6 位数字），未通过时禁用提交按钮
4. 本地 mock 校验验证码（测试码 `123456`）
5. 校验失败弹窗提示 `验证码错误`，3 秒后自动关闭（全屏完全禁交互）
6. 校验成功后进入「缘分墙 - 活跃状态 (普通用户版)」
7. 用户在缘分墙点击「再等等 / 感兴趣」，完成最小决策闭环（mock）

## 2. 版本范围（仅本轮）

包含：
1. 登录页 UI 与输入校验
2. mock 登录校验（不接后端）
3. 登录成功跳转到缘分墙活跃态
4. 登录失败弹窗（3 秒自动关闭，期间全屏完全禁交互）
5. 登录成功后将用户基础信息保存在内存（仅当前进程）
6. 缘分墙活跃态页面展示（仅成功路径，字段见 API Mock Contract）
7. 登录页协议/隐私入口点击后以不同 toast 占位提示（用户协议/隐私政策，1500ms 各自独立节流）
8. 缘分墙活跃态从本地 mock 池展示 1 条资料（mock 池最小 3 条），同一自然日内（含重新登录）保持同一对象
9. 登录与缘分墙 `loading` 当前默认最小时长均为 600ms（可按体验调整）
10. 缘分墙动作区：`再等等`、`感兴趣` 两个按钮
11. 今日推荐约束：用户当天仅查看 1 个推荐对象，不支持点击后刷新下一张卡片
12. `再等等` 行为：记录当天已选择，不刷新卡片，下一天再推荐
13. `感兴趣` 行为：走固定 mock 匹配规则（`profileId=="p_001"` 视为匹配成功）
14. 当天动作点击即锁定，不允许从 `再等等` 改选 `感兴趣`（或反向改选）
15. 锁定后再次点击动作按钮为静默忽略（不弹 toast）
16. `再等等` toast 文案固定为：`已为你安排明日推荐`
17. `感兴趣` 未匹配时 toast：`今日匹配已用完，请明日再来`
18. `感兴趣` 匹配成功时跳转消息栏占位页，返回后按当前 `dateKey` 回到缘分墙应展示卡片
19. 每天本地时间 00:00 自动重置“今日推荐对象与动作锁定”
20. App 进程被杀后动作锁定不保留（重新登录仍展示当天同一对象）
21. 用户停留缘分墙页面跨越 00:00 时，需立即重置并换到历史未推荐对象，不等待下次交互
22. `matched=true` 时会话标识按 `conversationId = profileId + "_" + dateKey` 生成
23. 同一用户在当前推荐循环内，已推荐过的对象不重复推荐
24. 当用户未推荐池耗尽时，不在当日重置循环历史；于下一天 00:00 重置循环历史并重新循环推荐
25. 长期历史仅在“卸载/清除应用数据”时清空
26. 当次日 00:00 进入新循环后，新循环首个推荐对象优先不与上一自然日最后展示对象相同
27. `login_submit_blocked` 仅在“合法 -> 非法”边沿触发；首次进入页面初始非法态不触发
28. “上一自然日最后展示对象”按“系统最后一次成功分配给该用户的推荐对象”定义（即使当时停留在消息占位页）
29. 若新循环可选池无法满足“首卡 != 上一日末卡”，允许降级为重复上一日末卡（不阻断主流程）
30. 消息占位页在 MVP 阶段不做跨 00:00 特殊处理；返回缘分墙时按当前日期规则立即重算并展示
31. 非 MVP 聊天组件可保留文件，但登录/缘分墙/消息占位主流程禁止 import 与路由跳转引用

不包含：
1. 真实聊天对话能力（仅消息页占位）
2. 举报/拉黑
3. 账号删除
4. 会员/IAP
5. 真实后端登录/验证码服务接入
6. 真实匹配服务（双方兴趣结果仅 mock）

## 3. 核心验收标准

1. 启动后默认进入登录页（吃苹果熊猫 Logo 版本）
2. 仅当手机号与验证码格式合法时，登录/注册按钮可点击（手机号仅校验 11 位数字）
3. 点击登录/注册后正确执行 mock 校验
4. 验证码错误时显示 `验证码错误` 弹窗，并在 3 秒后自动关闭（期间全屏完全禁交互）
5. 验证码为 `123456` 时登录成功并跳转到「缘分墙 - 活跃状态 (普通用户版)」
6. 登录成功后用户基础信息仅保存在内存，不做持久化
7. MVP 不使用登录鉴权 token；UI 必须使用 design tokens
8. 内存会话仅在 App 进程被杀后清空，前后台切换不清空
9. `login_submit_blocked` 事件按“合法 -> 非法”状态边沿触发一次；维持非法态期间不重复触发
10. 首次进入登录页（初始非法态）不触发 `login_submit_blocked`
11. UI 通过 `npm run ui:check`
12. 同一天内重新登录后，缘分墙仍展示当天同一推荐对象
13. 缘分墙点击 `再等等` 后，卡片不刷新，仍为当天推荐对象
14. 缘分墙点击 `感兴趣` 后，mock 命中双方感兴趣时可进入消息栏占位页
15. 同一天内动作点击即锁定，不允许改选
16. 动作锁定后再次点击按钮为静默忽略（不弹 toast）
17. 日期边界按设备本地自然日（00:00）计算，且到 00:00 自动重置
18. `再等等` toast 固定文案为：`已为你安排明日推荐`
19. `感兴趣` 未匹配时出现 toast：`今日匹配已用完，请明日再来`
20. 从消息栏占位页返回时，按当前日期规则立即重算并展示缘分墙当前应展示卡片
21. App 进程被杀后动作锁定不保留
22. 缘分墙页面停留跨越 00:00 时立即重置并切换到历史未推荐卡片
23. `matched=true` 导航到消息占位页时，`conversationId` 符合 `profileId + "_" + dateKey`
24. 同一用户不会再次看到当前循环内已推荐过的对象
25. 未推荐池耗尽后需等到下一天 00:00 重置循环历史再进入新循环
26. 长期历史仅在卸载/清除应用数据后清空
27. 次日 00:00 进入新循环后，首卡优先不等于上一自然日末卡；若无可选候选可允许重复
28. “上一自然日末卡”按“系统最后一次成功分配推荐对象”定义

## 4. 关键指标（本轮）

1. 登录流程完成率 >= 95%
2. 验证码错误弹窗触达率 = 100%
3. 登录成功后跳转缘分墙成功率 = 100%
4. 关键事件本地打印覆盖率 = 100%
5. 缘分墙动作点击事件本地打印覆盖率 = 100%

## 5. 风险与依赖

1. mock 规则与后续真实后端规则可能不一致，联调前需统一
2. 当前仅覆盖登录、缘分墙动作闭环与消息栏占位，非完整社交流程
3. 非 MVP 聊天组件可保留文件，但本轮不得接入主流程
