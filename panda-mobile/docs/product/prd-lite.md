# Panda Mobile PRD Lite

## 1. 背景与目标

`Panda Mobile` 首发目标是验证「轻社交匹配 + 即时聊天」闭环是否成立。  
首发版本仅聚焦最核心路径，不追求功能丰富度。

## 2. 版本范围（MVP）

包含：
1. 手机号 + 验证码登录
2. 缘分墙推荐卡片浏览与刷新
3. 单会话聊天（基础消息发送）
4. 结束连接
5. 举报/拉黑入口（可先占位）
6. 账号删除入口（可先占位）

不包含：
1. 复杂推荐策略配置后台
2. 多端同步与离线消息
3. 会员、虚拟币、IAP
4. 音视频通话

## 3. 目标用户

1. 希望低负担认识新朋友的年轻用户
2. 偏好轻量文本交流而非重社交关系链用户

## 4. 核心指标（首发）

1. 登录成功率 >= 95%
2. 从登录到首条消息发送完成率 >= 60%
3. 聊天发送成功率 >= 99%
4. 崩溃率 < 0.5%

## 5. 验收标准

1. 主流程可在真机完整走通：登录 -> 缘分墙 -> 聊天 -> 结束连接
2. 每个核心页面均覆盖 `loading/empty/error/content`
3. UI 必须通过 `npm run ui:check`
4. iOS 审核相关入口齐备：举报、拉黑、账号删除、隐私政策

## 6. 依赖与风险

1. 后端接口未就绪时使用 mock service，联调时逐步切换真实 API
2. 审核风险集中在 UGC 合规能力不完整
3. 字段变更风险通过 API 合同与版本号控制
