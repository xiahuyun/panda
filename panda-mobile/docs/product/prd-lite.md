# Panda Mobile PRD Lite (V0.5)

## 1. 目标

本版本仅验证最小可用主流程：
1. 用户进入「小熊交友 - 吃苹果熊猫 Logo 登录页」
2. 用户输入手机号与验证码并点击「登录/注册」
3. 前端先做输入合法性拦截（手机号 11 位数字，不限号段；验证码 6 位数字），未通过时禁用提交按钮
4. 本地 mock 校验验证码（测试码 `123456`）
5. 校验失败弹窗提示 `验证码错误`，3 秒后自动关闭（全屏完全禁交互）
6. 校验成功后进入「缘分墙 - 活跃状态 (普通用户版)」

## 2. 版本范围（仅本轮）

包含：
1. 登录页 UI 与输入校验
2. mock 登录校验（不接后端）
3. 登录成功跳转到缘分墙活跃态
4. 登录失败弹窗（3 秒自动关闭，期间全屏完全禁交互）
5. 登录成功后将用户基础信息保存在内存（仅当前进程）
6. 缘分墙活跃态页面展示（仅成功路径，字段见 API Mock Contract）
7. 登录页协议/隐私入口点击后以不同 toast 占位提示（用户协议/隐私政策，1500ms 各自独立节流）
8. 缘分墙活跃态从本地 mock 池随机展示 1 条资料（mock 池最小 3 条）
9. 登录与缘分墙 `loading` 当前默认最小时长均为 600ms（可按体验调整）

不包含：
1. 聊天页
2. 再等等/感兴趣策略
3. 举报/拉黑
4. 账号删除
5. 会员/IAP
6. 真实后端登录/验证码服务接入

## 3. 核心验收标准

1. 启动后默认进入登录页（吃苹果熊猫 Logo 版本）
2. 仅当手机号与验证码格式合法时，登录/注册按钮可点击（手机号仅校验 11 位数字）
3. 点击登录/注册后正确执行 mock 校验
4. 验证码错误时显示 `验证码错误` 弹窗，并在 3 秒后自动关闭（期间全屏完全禁交互）
5. 验证码为 `123456` 时登录成功并跳转到「缘分墙 - 活跃状态 (普通用户版)」
6. 登录成功后用户基础信息仅保存在内存，不做持久化
7. MVP 不使用登录鉴权 token；UI 必须使用 design tokens
8. 内存会话仅在 App 进程被杀后清空，前后台切换不清空
9. `login_submit_blocked` 事件不去重，输入非法即触发
10. UI 通过 `npm run ui:check`

## 4. 关键指标（本轮）

1. 登录流程完成率 >= 95%
2. 验证码错误弹窗触达率 = 100%
3. 登录成功后跳转缘分墙成功率 = 100%
4. 关键事件本地打印覆盖率 = 100%

## 5. 风险与依赖

1. mock 规则与后续真实后端规则可能不一致，联调前需统一
2. 当前仅覆盖登录与进入缘分墙，非完整社交流程
3. 非 MVP 聊天组件需从主分支移除，避免误接入
