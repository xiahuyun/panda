# Panda Mobile PRD Lite (V0.2)

## 1. 目标

本版本仅验证最小可用主流程：
1. 用户进入「小熊交友 - 吃苹果熊猫 Logo 登录页」
2. 用户输入手机号与验证码并点击「登录/注册」
3. 本地 mock 校验验证码（6 位，测试码 `123456`）
4. 校验失败弹窗提示 `验证码错误`，3 秒后自动关闭
5. 校验成功后进入「缘分墙 - 活跃状态 (普通用户版)」

## 2. 版本范围（仅本轮）

包含：
1. 登录页 UI 与输入校验
2. mock 登录校验（不接后端）
3. 登录成功跳转到缘分墙活跃态
4. 登录失败弹窗（3 秒自动关闭）
5. 缘分墙活跃态页面展示（字段见 API Mock Contract）

不包含：
1. 聊天页
2. 再等等/感兴趣策略
3. 举报/拉黑
4. 账号删除
5. 会员/IAP
6. 真实后端登录/验证码服务接入

## 3. 核心验收标准

1. 启动后默认进入登录页（吃苹果熊猫 Logo 版本）
2. 点击登录/注册后正确执行 mock 校验
3. 验证码错误时显示 `验证码错误` 弹窗，并在 3 秒后自动关闭
4. 验证码为 `123456` 时登录成功并跳转到「缘分墙 - 活跃状态 (普通用户版)」
5. MVP 不使用 token，不持久化登录状态
6. UI 通过 `npm run ui:check`

## 4. 关键指标（本轮）

1. 登录流程完成率 >= 95%
2. 验证码错误弹窗触达率 = 100%
3. 登录成功后跳转缘分墙成功率 = 100%

## 5. 风险与依赖

1. mock 规则与后续真实后端规则可能不一致，联调前需统一
2. 当前仅覆盖登录与进入缘分墙，非完整社交流程
3. 非 MVP 聊天组件需从主分支移除，避免误接入
