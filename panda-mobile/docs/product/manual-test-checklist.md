# Panda Mobile 手测清单 (MVP)

更新时间: 2026-03-12
适用范围: D1-D5(P1) MVP 主流程

## 1. 登录页

1. 启动默认进入登录页（`pages/auth/login`）。【已自动确认】
2. 手机号非 11 位数字时，提交按钮禁用。【已端上回归通过（HBuilderX，2026-03-11）】
3. 验证码非 6 位数字时，提交按钮禁用。【已端上回归通过（HBuilderX，2026-03-11）】
4. 输入状态从合法变为非法时，控制台出现 `login_submit_blocked` 本地事件日志（持续非法不重复触发；首次进入页面初始非法态不触发）。【已自动确认】

## 2. 登录提交流程

1. 输入合法手机号 + `123456`，点击登录/注册后进入提交态。【已自动确认】
2. 约 600ms 后登录成功，跳转 `pages/affinity/active`。【已自动确认】
3. 输入合法手机号 + 非 `123456` 验证码，显示 `验证码错误` 弹窗。【已自动确认】
4. 错误弹窗 3 秒后自动关闭。【已自动确认】
5. 错误弹窗期间无法点击页面、无法编辑输入、无法重复提交、返回键被拦截。【已端上验证通过（HBuilderX）】

## 3. 协议与隐私入口

1. 点击用户协议，显示 toast：`用户协议建设中`。【已自动确认】
2. 点击隐私政策，显示 toast：`隐私政策建设中`。【已自动确认】
3. 用户协议/隐私政策各自 1500ms 节流，连续点击不重复弹出。【已自动确认】

## 4. 缘分墙活跃态

1. 页面首次进入先显示 loading（约 600ms）。【已自动确认】
2. loading 后显示资料卡片与推荐理由。【已自动确认】
3. 资料来自本地 mock 池（最小 3 条），并按同日稳定策略选取。 【已自动确认（`mock/affinity-profiles.mock.json` 为真源，经脚本生成 `mock/affinity-profiles.mock.uts` 后由 service 读取）】

5. 头像统一占位 `/static/logo.png`。【已自动确认】

## 7. HBuilderX 手测执行步骤

1. 使用 HBuilderX 打开目录 `panda-mobile`。
2. 选择运行到模拟器或真机，进入登录页 `pages/auth/login`。
3. 输入合法手机号 + 非 `123456` 验证码，触发 `验证码错误` 弹窗。
4. 弹窗显示期间尝试:
   - 点击页面任意区域
   - 编辑手机号/验证码输入框
   - 重复点击登录按钮
   - 触发系统返回操作
5. 预期结果:
   - 全部交互被阻断
   - 返回操作被拦截
   - 3 秒后弹窗自动关闭并恢复交互
6. 执行完成后，将第 2.5 条状态改为【通过】或【不通过（附现象）】。

## 8. D5 P1 新增待测项（已实现，已端上回归通过）

1. 点击 `再等等` 后：
   - 当前卡片不刷新
   - 当天动作锁定，不允许改选
   - toast 固定为 `已为你安排明日推荐`
   - 状态：【已自动确认（代码路径）】
2. 点击 `感兴趣` 后：
   - 按固定规则匹配（`profileId=="p_001"` 为 `matched=true`）
   - `matched=false` 时 toast：`今日匹配已用完，请明日再来`
   - `matched=true` 时跳转消息栏占位页
   - `conversationId` 按 `profileId + "_" + dateKey` 生成
   - 状态：【已自动确认（代码路径）】
3. 当天动作已锁定时再次点击任一动作：
   - 静默忽略（不弹 toast）
   - 记录 `affinity_daily_action_locked` 本地事件
   - 状态：【已自动确认（代码路径）】
4. 同一自然日重新登录：
   - 仍展示当天同一推荐对象
   - 不应回退到当前循环内已推荐对象
   - 状态：【已自动确认（代码路径）】
5. 从消息栏占位页返回：
   - 回到缘分墙并按当前 `dateKey` 立即重算卡片
   - 重算结果满足当日规则（含同日稳定/跨日重置）
   - 状态：【已端上验证通过（HBuilderX，2026-03-12）】
6. App 进程被杀后再次登录（同一天）：
   - 动作锁定不保留（可重新选择动作）
   - 推荐对象仍为当天同一对象
   - 状态：【已端上验证通过（HBuilderX，2026-03-12）】
7. 日期边界验证：
   - 到设备本地时间 00:00 自动重置次日推荐状态与动作锁定
   - 页面停留中到点应立即重置并换到历史未推荐卡片（不等待下次交互/onShow）
   - 状态：【已端上验证通过（HBuilderX，2026-03-12）】
8. 推荐池耗尽与历史周期验证：
   - 当未推荐池耗尽时，当日保持当前卡片，不立即清空循环历史
   - 下一天 00:00 重置循环历史并重新循环
   - 新循环首个推荐对象优先不应等于上一自然日最后展示对象（无可选候选可降级重复）
   - “上一自然日最后展示对象”按系统最后一次成功分配推荐对象定义
   - 不卸载/不清除应用数据时，长期历史持续保留
   - 状态：【已端上验证通过（HBuilderX，2026-03-12）】
