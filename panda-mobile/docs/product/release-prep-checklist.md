# Panda Mobile 封版与内测准备清单 (D5 P1)

更新时间: 2026-03-12  
适用范围: D5 P1（登录 -> 缘分墙动作闭环 -> 消息占位）

## 1. 封版范围确认

1. 本轮封版范围仅包含：登录页 + 缘分墙活跃态（mock 数据）+ 再等等/感兴趣动作 + 消息占位页。
2. 不包含：真实聊天能力、真实后端验证码服务、会员/IAP、真实匹配服务。
3. 封版基线文档：
   - `docs/product/prd-lite.md`
   - `docs/product/ui-schema.stitch.json`
   - `docs/product/manual-test-checklist.md`

## 2. 提交前门禁

1. `npm run mock:sync` 通过。
2. `npm run ui:check` 通过（token scan + script setup scan）。
3. `main` 分支 GitHub Actions 全绿。

## 3. HBuilderX 打包前配置检查

1. 打开 `manifest.json`，确认：
   - `appid`：`__UNI__8AD3E01`
   - `versionName`：`1.0.0`
   - `versionCode`：`100`
2. 确认应用名称、图标、启动图、隐私政策配置满足内测要求。
3. 确认测试环境说明：当前为 mock 登录（测试码 `123456`）。

## 4. HBuilderX 内测包执行步骤

1. 使用 HBuilderX 打开 `panda-mobile`。
2. 选择目标平台（iOS 内测优先）并执行云打包/本地打包。
3. 安装到测试设备后执行最小验收路径：
   - 启动进入登录页
   - 非法输入禁用提交（含手机号/验证码超长）
   - 错码弹窗 3 秒自动关闭且全屏禁交互
   - 正确码 `123456` 登录并跳转缘分墙
   - 同一自然日重新登录仍展示同一推荐对象
   - 当前循环内已推荐对象不再重复推荐给同一用户
   - 未推荐池耗尽时当日保持当前卡片，下一天 00:00 重置循环历史并重新循环
   - 次日 00:00 进入新循环后，首个推荐对象优先不等于上一自然日最后展示对象（无可选候选可降级重复）
   - “上一自然日最后展示对象”按系统最后一次成功分配推荐对象定义
   - 长期历史仅在卸载/清除应用数据后清空
   - 点击 `再等等` 后不刷新卡片，动作当天锁定
   - 当天动作锁定后再次点击任一动作为静默忽略（无 toast）
   - 点击 `感兴趣` 后按固定规则匹配（`profileId=="p_001"` 为 true）
   - `matched=true` 时 `conversationId` 按 `profileId + "_" + dateKey` 生成
   - `matched=false` 时 toast：`今日匹配已用完，请明日再来`
   - `matched=true` 时进入消息占位页，并在返回缘分墙时按当前 `dateKey` 立即重算卡片
   - 消息占位页在 MVP 阶段不处理跨 00:00 特殊逻辑
   - 首次进入登录页初始非法态不触发 `login_submit_blocked`
   - App 进程被杀后重新登录（同一天）动作锁定不保留
   - 页面停留跨越 00:00 时立即重置并换到历史未推荐卡片（不等待下次交互/onShow）
4. 记录内测结果与问题列表（如有）。
