# Panda Mobile 状态矩阵 (V0.6)

## 1. 登录页（吃苹果熊猫 Logo）

| 状态 | 触发条件 | UI 表现 | 用户操作 |
| --- | --- | --- | --- |
| `content` | 初始进入页面 | 显示手机号、验证码输入框与登录/注册按钮 | 输入 |
| `invalid` | 手机号非 11 位数字或验证码非 6 位数字 | 登录/注册按钮禁用 | 继续输入修正 |
| `loading` | 点击登录/注册后 | 按钮提交中（当前默认最短展示 600ms，可按体验调整） | 等待 mock 返回 |
| `error_modal` | mock 校验失败 | 弹窗文案 `验证码错误`，全屏锁定交互（不可输入/不可返回/不可点击） | 等待 3 秒自动关闭 |
| `success` | mock 校验通过（验证码=123456） | 登录成功并写入内存会话 | 跳转缘分墙活跃态 |

## 2. 缘分墙 - 活跃状态 (普通用户版)

| 状态 | 触发条件 | UI 表现 | 用户操作 |
| --- | --- | --- | --- |
| `loading` | 登录成功后加载 mock 数据 | 页面骨架或加载中提示（当前默认最短展示 600ms，可按体验调整） | 等待 |
| `active` | mock 数据加载成功 | 展示活跃态卡片 + 会员入口（灰态展示，不可点击） | 浏览 |
| `decision_wait` | 点击 `再等等` | 保持当前卡片，不刷新；toast：`已为你安排明日推荐` | 等待次日 |
| `decision_interested` | 点击 `感兴趣` | 记录当日动作并执行 mock 匹配 | 等待结果 |
| `match_navigate` | `感兴趣` 且 `matched=true` | 跳转消息栏占位页 | 查看占位页 |
| `match_stay` | `感兴趣` 且 `matched=false` | 停留当前卡片，不刷新，并 toast：`今日匹配已用完，请明日再来` | 停留 |
| `action_locked` | 当天已点击任一动作 | 动作锁定，不可改选；再次点击静默忽略（无 toast） | 等待次日 |
| `day_reset` | 页面停留中到本地时间 00:00 | 立即重置当日推荐与动作锁定，并切换到历史未推荐卡片 | 继续浏览 |
| `pool_exhausted_hold` | 当前循环无未推荐对象 | 保持当前卡片，不当日换新 | 等待下一天 00:00 重置循环历史并优先进入首卡不等于昨日末卡的新循环（无可选候选可降级重复） |

## 3. 消息栏占位页（MVP）

| 状态 | 触发条件 | UI 表现 | 用户操作 |
| --- | --- | --- | --- |
| `placeholder` | 从缘分墙 `matched=true` 进入 | 显示“消息功能建设中”占位内容 | 返回缘分墙并按当前 `dateKey` 立即重算卡片 |

## 4. 状态规则（放宽版）

1. 不强制每个页面都具备 `loading/empty/error/content` 四态。
2. 每个页面只定义业务必要状态即可。
3. 缘分墙本轮不实现 `error` 状态分支。
4. 登录内存会话仅在 App 进程被杀后清空，前后台切换不清空。
5. 协议/隐私 toast 1500ms 内不重复弹出，且两者各自独立节流。
6. 缘分墙同一自然日仅展示 1 张推荐卡片（含重新登录）；点击动作后不刷新下一张。
7. 同一自然日动作点击即锁定，不允许 `wait` 与 `interested` 互相改选。
8. 动作锁定仅保存在当前 App 进程；App 进程被杀后不保留。
9. 日期边界按设备本地自然日（00:00）计算，并在 00:00 自动重置当日推荐与动作锁定。
10. 若用户停留在缘分墙页面，00:00 到点后立即执行重置并换到历史未推荐卡片，不等待下次交互或 `onShow`。
11. 推荐对象按用户循环历史去重：同一用户当前循环内已推荐过的 `profileId` 后续不再推荐。
12. 当未推荐池耗尽时，保持当前卡片到次日 00:00；次日 00:00 重置循环历史并重新循环。
13. 次日 00:00 进入新循环后，首个推荐对象优先不等于上一自然日最后展示对象。
14. “上一自然日最后展示对象”按系统最后一次成功分配推荐对象定义。
15. 若新循环无可选候选满足“首卡 != 上一日末卡”，允许降级重复上一日末卡。
16. 长期历史仅在卸载/清除应用数据时清空。
17. 消息占位页在 MVP 阶段不处理跨 00:00 特殊逻辑；返回缘分墙时按当前 `dateKey` 立即重算。
18. `login_submit_blocked` 在输入“合法 -> 非法”边沿触发；首次进入页面初始非法态不触发。
