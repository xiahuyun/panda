# Panda Mobile 状态矩阵 (V0.2)

## 1. 登录页（吃苹果熊猫 Logo）

| 状态 | 触发条件 | UI 表现 | 用户操作 |
| --- | --- | --- | --- |
| `content` | 初始进入页面 | 显示手机号、验证码输入框与登录/注册按钮 | 输入并提交 |
| `loading` | 点击登录/注册后 | 按钮提交中 | 等待 mock 返回 |
| `error_modal` | mock 校验失败 | 弹窗文案 `验证码错误` | 等待 3 秒自动关闭 |
| `success` | mock 校验通过（验证码=123456） | 登录成功 | 跳转缘分墙活跃态 |

## 2. 缘分墙 - 活跃状态 (普通用户版)

| 状态 | 触发条件 | UI 表现 | 用户操作 |
| --- | --- | --- | --- |
| `loading` | 登录成功后加载 mock 数据 | 页面骨架或加载中提示 | 等待 |
| `active` | mock 数据加载成功 | 展示活跃态卡片 | 浏览 |
| `error` | mock 数据异常 | 错误提示 + 重试入口 | 重试 |

## 3. 状态规则（放宽版）

1. 不强制每个页面都具备 `loading/empty/error/content` 四态。
2. 每个页面只定义业务必要状态即可。
