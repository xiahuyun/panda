# Panda Mobile 导航与信息架构 (V0.6)

## 1. 页面清单（MVP）

1. `LoginPage`：小熊交友 - 吃苹果熊猫 Logo 登录页
2. `AffinityWallActivePage`：缘分墙 - 活跃状态 (普通用户版)
3. `MessagePlaceholderPage`：消息栏占位页（MVP）

## 2. 路由关系

1. 启动 -> `LoginPage`
2. `LoginPage` 登录成功 -> `AffinityWallActivePage`
3. `LoginPage` 登录失败 -> 停留 `LoginPage`（验证码错误弹窗）
4. `AffinityWallActivePage` 点击 `感兴趣` 且 mock 匹配成功 -> `MessagePlaceholderPage`
5. `MessagePlaceholderPage` 返回 -> `AffinityWallActivePage`（按当前 `dateKey` 立即重算并展示当前应展示卡片）

## 3. 导航规则

1. 登录成功后清理登录页回退栈，避免返回未登录页。
2. MVP 不接入真实聊天页与设置页导航（仅保留消息栏占位页）。
3. MVP 不持久化登录状态，每次启动默认进入登录页。
4. 缘分墙页 `membership_entry` 灰态展示，不可点击跳转。
5. 登录页协议/隐私入口点击后使用 toast 占位，不跳转外部页面。
6. 协议入口 toast：`用户协议建设中`；隐私入口 toast：`隐私政策建设中`。
7. 内存会话仅在 App 进程被杀后清空，前后台切换不清空。
8. 协议/隐私 toast 在 1500ms 内不重复弹出，且两者各自独立节流。
9. 缘分墙同一自然日仅展示 1 个推荐对象（含重新登录），点击 `再等等/感兴趣` 后不刷新下一张卡片。
10. `再等等`：记录当天已选择，下一天再推荐，toast 固定为 `已为你安排明日推荐`。
11. `感兴趣`：mock 匹配成功时允许跳转消息栏占位页，`conversationId` 按 `profileId + "_" + dateKey` 生成。
12. 动作点击即锁定，当天不允许改选。
13. 动作锁定后再次点击为静默忽略（不弹 toast）。
14. `感兴趣` 未匹配时 toast：`今日匹配已用完，请明日再来`。
15. 日期边界按设备本地自然日（00:00）计算，并在 00:00 自动重置当日推荐与动作锁定。
16. 页面停留中跨越 00:00 时立即重置并换到历史未推荐卡片，不等待下次交互/onShow。
17. App 进程被杀后动作锁定不保留。
18. 推荐对象按用户循环历史去重：同一用户当前循环内已推荐对象后续不再推荐。
19. 若未推荐池耗尽，当日保持当前卡片；下一天 00:00 重置循环历史后重新循环。
20. 次日 00:00 进入新循环后，首个推荐对象优先不等于上一自然日最后展示对象。
21. 长期历史仅在卸载/清除应用数据时清空。
22. 消息占位页在 MVP 阶段不处理跨 00:00 特殊逻辑。
23. “上一自然日最后展示对象”按系统最后一次成功分配推荐对象定义。
24. 若新循环无可选候选可满足“首卡 != 上一日末卡”，允许降级重复上一日末卡。
25. 非 MVP 聊天组件保留文件但未接入：主流程页面禁止 import，主流程路由禁止跳转到非 MVP 聊天路由。

## 4. 信息架构（MVP）

1. 账户域：手机号、验证码、登录结果
2. 缘分墙域：活跃态卡片展示、动作决策（再等等/感兴趣）
3. 消息域：占位页展示（无真实会话）

## 5. 后续扩展位（未启用）

1. 真实聊天域（消息列表/发送/会话管理）
2. 举报/拉黑域
3. 账号管理域
